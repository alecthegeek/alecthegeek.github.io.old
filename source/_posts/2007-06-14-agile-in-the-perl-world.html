--- layout: post title: Agile in the Perl world tags: - Perl status: publish type: post published: true meta: {} --- Or should that be Perl in an Agile World? There was a discussion tonight at Melbourne Perl Mongers on using Agile practices whilst Perl programming. To my delight the answer from attendees was 'Yes we do that every day'! Specifically there was a discussion around:
  • TDD frameworks
  • Refactoring
  • Continuous integration and how important is is to have an SCM tool that makes developer branches (and merging) easy
  • Perl Mock Objects for testing
  • How to apply TDD to code that is not OO or even a proper module
There is a Yahoo Group on Perl Extreme Programming and an online book
