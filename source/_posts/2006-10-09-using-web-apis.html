--- layout: post title: Using Web API's tags: - Web status: publish type: post published: true meta: {} --- Last Thursday I went to the Web Standards Group meeting here in Melbourne to see the presentation given by Jeremy Keith on the "Joy of API ". The thrust of the talk was to outline the power of web API's, for web sites such as flickr , Amazon , Google Maps etc, and how it can be used to enhance your own web sites. The take away messages were:
  • There is a lot a information and functionality you can add to your web site, that is actually provided by other people's web API's
  • REST is better (simpler) than SOAP
  • Microformats exist to make your content parse-able from other sites, this can be considered as a poor man's API enabling technology
  • If you want to publish your own Web APIm then base it on exisiting practice (i.e copy someone else's)
